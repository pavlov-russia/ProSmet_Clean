# ADR-006 · Harness отделен от продуктового описания

Дата: 2026-05-19  
Статус: принято

## Контекст

В черновике `AGENTS.md` и `CLAUDE.md` фактически пересказывали README продукта. Такой формат расходует контекст и слабо управляет поведением агента.

## Решение

Harness-файлы описывают роль агента, правила работы, разрешения, запреты, маршруты контекста и quality gates. Продуктовый смысл вынесен в `Vision.md`, `Scope.md` и `docs/`.

## Последствия

- `AGENTS.md` становится общим neutral harness для AI-агентов.
- `CLAUDE.md` импортирует `AGENTS.md` и добавляет только Claude Code-специфику.
- `.claude/settings.json` хранит enforceable permissions.
- Большие процедуры выносятся в `harness/skills/` или `.claude/rules/`.
