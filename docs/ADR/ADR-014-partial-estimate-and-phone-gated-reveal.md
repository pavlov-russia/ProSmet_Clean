# ADR-014 · Partial estimate и phone-gated full reveal

Дата: 2026-05-19  
Статус: принято  
Связано: `docs/ConceptualDesign.md`, `docs/SolutionDesign.md`, `docs/ADR/ADR-009-phone-gate-and-consent.md`

## Контекст

Если клиент не знает часть параметров, продукт не должен показывать тупик. Для B2C-конверсии важнее дать честный предварительный результат: что уже посчитано, чего не хватает и что будет уточнено.

Также архитектор уточнил MVP-2: клиент сначала вводит параметры, видит часть сметы и замыленные блоки, затем оставляет телефон и получает полное раскрытие.

## Решение

Calculation engine поддерживает partial estimate.

Правила:

- известные строки считаются deterministic engine;
- неизвестные строки попадают в смету со статусом `insufficient_data`;
- итог помечается как частичный, если есть неизвестные amount-bearing blocks;
- UI показывает "недостаточно данных", а не скрытую ошибку;
- preview до телефона может быть замыленным;
- полное раскрытие, PDF и клиентская ссылка требуют phone-gate, если он включен.

Policy engine может вернуть `auto_publish` с:

- `publicationMode = full`;
- `publicationMode = partial`.

Partial publication допустима только если tenant разрешил это в policy profile и клиент явно видит неполноту.

## Последствия

- Missing fields больше не всегда означают отказ от расчета.
- `reject_as_incomplete` остается только для полностью непригодного ввода.
- Client link/PDF должны ясно показывать неполные блоки.
- Quality gates должны проверять, что partial total не выдается за финальный полный расчет.
