# ADR-015 · РФ data residency и путь развертывания

Дата: 2026-05-19  
Статус: принято  
Связано: `docs/SolutionDesign.md`, `docs/Architecture.md`

## Контекст

Разработка ProSmet может идти локально или на VPS. Но после пилотной проверки продукт должен быть развернут на БД и серверах, совместимых с требованиями РФ к хранению и обработке персональных данных.

## Решение

Разделяем окружения:

- local — разработка на тестовых данных;
- VPS/dev — инженерная проверка и демо без реальных ПД, если не принято отдельное решение;
- RF production — БД, object storage, logs и backups в российской инфраструктуре.

Перед публичным запуском требуется отдельный deployment milestone:

- выбрать РФ-провайдера;
- поднять PostgreSQL в РФ;
- поднять object storage для PDF в РФ;
- настроить логи и backup в РФ;
- подтвердить secrets policy;
- подтвердить PII retention/deletion policy;
- проверить, что AI payload не содержит ПД и денег.

## Последствия

- Реальные ПД не должны попадать в неподтвержденное окружение.
- Архитектура должна предусматривать миграцию с local/VPS на RF production.
- DevOps получает отдельный quality gate по РФ-контуру перед production.
