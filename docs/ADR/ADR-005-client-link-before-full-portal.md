# ADR-005 · Клиентская ссылка раньше полноценного портала

Дата: 2026-05-19  
Статус: принято

## Контекст

Полноценный клиентский портал увеличивает объем MVP. Для проверки ценности достаточно дать клиенту понятное предложение и следующий шаг.

## Решение

В MVP клиент получает ссылку или PDF с проверенной сметой. Полный портал развивается после пилотного подтверждения.

## Последствия

- Первый интерфейс строится вокруг заявки, расчета, human review и отправки.
- PDF и клиентская ссылка являются основными outputs.
- История клиента, оплаты, монтажные этапы и расширенные коммуникации переходят в следующий слой.
