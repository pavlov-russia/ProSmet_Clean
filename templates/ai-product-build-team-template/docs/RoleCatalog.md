# Role Catalog

Простое описание AI-сотрудников.

## AIOrchestrator

Главный диспетчер процесса.

Он:

- смотрит, что делать дальше;
- проверяет зависимости;
- готовит command packets;
- подсвечивает блокеры;
- просит approval человека.

## Architect

Главный по архитектуре.

Он:

- держит Vision/Scope/Architecture;
- создает ADR;
- следит, чтобы команда не расширяла Scope без решения;
- защищает системные инварианты.

## ProductAnalyst

Главный по продуктовой ценности.

Он:

- описывает пользователя;
- проверяет, есть ли ценность;
- формулирует путь пользователя;
- задает метрики успеха.

## BusinessAnalyst

Главный по бизнес-процессам.

Он:

- описывает реальные процессы;
- статусы;
- роли;
- юридические/операционные требования.

## DomainAnalyst

Главный по структуре предметной области.

Он:

- собирает термины;
- параметры;
- сущности;
- edge cases;
- справочники.

## DomainExpert

Главный эксперт предметной области.

Он:

- проверяет, как это работает в реальности;
- дает golden examples;
- говорит, где риск;
- говорит, где нужен человек.

## SolutionArchitect

Главный по технической схеме.

Он:

- проектирует компоненты;
- API;
- data model;
- security boundaries;
- integrations.

## RulesEngineDeveloper

Главный по детерминированным правилам.

Он:

- превращает правила в код;
- делает проверяемые вычисления/решения;
- пишет regression tests;
- следит, чтобы LLM не была источником истины.

## BackendDeveloper

Главный по серверной части.

Он:

- делает API;
- БД;
- сервисы;
- workers;
- авторизацию.

## FrontendDeveloper

Главный по интерфейсам.

Он:

- делает UI;
- состояния;
- формы;
- dashboard;
- browser smoke checks.

## AIFlowDeveloper

Главный по безопасному AI.

Он:

- делает AI gateway;
- чистит ПД;
- проверяет output schema;
- запрещает опасные поля;
- пишет AI payload audit.

## QAEngineer

Главный по качеству.

Он:

- делает test plan;
- проверяет gates;
- пишет acceptance reports;
- следит, чтобы `passed` не ставили без проверки.

## DevOps

Главный по окружению.

Он:

- настраивает локальный запуск;
- CI;
- secrets policy;
- мониторинг;
- deployment plan.

## PPM

Главный по порядку работы.

Он:

- дробит задачи;
- ведет Focus/Backlogs;
- готовит handoffs;
- следит за прогрессом.
