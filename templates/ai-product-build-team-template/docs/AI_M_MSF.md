# AI_M_MSF

Методология подготовки продукта с AI-командой.

## Документная цепочка

```text
Vision -> Scope -> Conceptual Design -> Solution Design -> Architecture -> ADR -> Tasks -> Evidence
```

## Vision

Vision отвечает:

- зачем существует продукт;
- кто пользователи;
- какую боль решаем;
- какой образ будущего строим;
- какой долгосрочный горизонт.

Vision не режет мечту. Ограничения живут в Scope.

## Scope

Scope отвечает:

- что входит в MVP;
- что не входит;
- какие пользователи входят;
- какие сценарии входят;
- какие данные нужны;
- какие критерии готовности.

## Conceptual Design

Conceptual Design отвечает:

- какие роли пользователей;
- какие потоки;
- какие сущности;
- какой lifecycle;
- где человек принимает решения;
- где AI может помогать;
- где AI запрещен.

## Solution Design

Solution Design отвечает:

- какие компоненты есть;
- какие API нужны;
- какие данные храним;
- какие workers/adapters нужны;
- какие границы безопасности;
- какие тесты подтверждают систему.

## ADR

ADR фиксирует архитектурное решение:

```text
Context -> Decision -> Consequences
```

ADR нужен, чтобы команда не переоткрывала уже принятые развилки.
